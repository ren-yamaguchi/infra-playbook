## 2. 目的・概要

### 2-1. 目的

本手順書では、AWS 上に VPC・サブネット・インターネットゲートウェイ・ルートテーブル・EC2 インスタンスを構築する。
構築後は、ローカル PC から EC2 インスタンスへ SSH 接続できる状態を目指す。

なお、本手順書では各 AWS リソースを「住宅構築」の例えと紐づけながら説明する。

| AWSリソース | 住宅構築での例え |
|------------|-----------------|
| VPC の作成 | 家自体を作る |
| サブネットの作成 | 家の中を仕切って部屋を作る |
| インターネットゲートウェイ（IGW）の作成 | 玄関を作る |
| IGW のアタッチ | 玄関と部屋をつなぐ廊下を作る |
| ルーティングの設定 | 玄関までの行き方を教える |
| EC2 の作成 | 部屋の中に人（サーバ）を配置する |

### 2-2. 構成概要（アーキテクチャ）

```
Internet
   |
[IGW]
   |
[VPC: 10.1.0.0/16]
   └── [Public Subnet: 10.1.0.0/18]
            └── [EC2: Amazon Linux 2023]
```

### 2-3. 完成イメージ（ゴール定義）

- [ ] VPC（10.1.0.0/16）が作成されている
- [ ] サブネット（10.1.0.0/18）が作成されている
- [ ] インターネットゲートウェイが作成され、VPC にアタッチされている
- [ ] ルートテーブルに `0.0.0.0/0 → IGW` のルートが設定されている
- [ ] EC2 インスタンスが起動し、パブリック IP が割り当てられている
- [ ] ローカル PC から EC2 インスタンスに SSH ログインできる

## 4. 構築手順（詳細）

> **注意事項**
> - コマンド中・設定値中の `<山カッコ>` は自分の環境の値に置き換えること
> - `<任意の名前>` は、自分や他のメンバーと区別できる名前を決めること（例：`my-vpc-01`、`taro-vpc` など）

------------------------------

### Step 1：VPC の作成

**目的：** VPC を作成し、自分専用のプライベートネットワーク空間（家自体）を用意する。

#### 操作手順

1. AWS マネジメントコンソールにログインする。
2. 画面上部の検索欄に「VPC」と入力し、VPC サービスを選択する。
3. 左メニューから「VPC」を選択し、画面右上の「VPC を作成」をクリックする。
4. 以下の項目を入力する（その他はデフォルトのまま）。
   - **名前タグ**：`<任意の名前>`
   - **IPv4 CIDR**：`10.1.0.0/16`
5. 画面右下の「VPC を作成」をクリックする。

> **確認：** VPC 一覧に作成した VPC が表示され、ステータスが「使用可能（Available）」になっていることを確認する。

------------------------------

### Step 2：サブネットの作成

**目的：** VPC 内を区切ってサブネット（部屋）を作成し、EC2 インスタンスを配置できる場所を確保する。

#### 操作手順

1. VPC 画面の左メニューから「サブネット」を選択する。
2. 画面右上の「サブネットを作成」をクリックする。
3. **VPC ID** の入力欄で、Step 1 で作成した VPC を選択する。
4. 以下の項目を入力する。
   - **サブネット名**：`<任意の名前>`
   - **アベイラビリティーゾーン**：`ap-northeast-1a`（ログイン時に選択したリージョンに合わせる）
   - **IPv4 サブネット CIDR ブロック**：`10.1.0.0/18`
5. 画面右下の「サブネットを作成」をクリックする。

> **補足：** サブネットマスクは、VPC の `/16` を 4 人で分割するため、ホスト部から 2 ビット分ずらして `/18` となる。第 3 オクテットの値はメンバー間で重複しないように決める。

> **確認：** サブネット一覧に作成したサブネットが表示され、ステータスが「使用可能（Available）」になっていることを確認する。

------------------------------

### Step 3：インターネットゲートウェイの作成とアタッチ

**目的：** VPC とインターネットを接続するためのインターネットゲートウェイ（玄関）を作成し、VPC にアタッチ（廊下を通す）する。

#### 操作手順

**【3-1】インターネットゲートウェイの作成**

1. VPC 画面の左メニューから「インターネットゲートウェイ」を選択する。
2. 画面右上の「インターネットゲートウェイの作成」をクリックする。
3. 以下の項目を入力する。
   - **名前タグ**：`<任意の名前>_igw`
4. 画面右下の「インターネットゲートウェイの作成」をクリックする。

**【3-2】VPC へのアタッチ**

1. 作成したインターネットゲートウェイを検索欄で検索し、左の □ にチェックを入れる。
2. 画面右上の「アクション ▼」をクリックし、「VPC にアタッチ」を選択する。
3. 「使用可能な VPC」の検索欄に Step 1 で作成した VPC 名を入力し、選択する。
4. 画面右下の「インターネットゲートウェイのアタッチ」をクリックする。

> **確認：** インターネットゲートウェイの状態が「Attached」になっていることを確認する。

------------------------------

### Step 4：ルーティングの設定

**目的：** サブネット内のコンピュータがインターネットゲートウェイ経由で外部と通信できるように、ルートテーブルに経路（玄関までの行き方）を設定する。

#### 操作手順

1. VPC 画面の左メニューから「サブネット」を選択する。
2. 検索欄で Step 2 で作成したサブネットを検索し、左の □ にチェックを入れる。
3. 画面下部に表示される情報から「ルートテーブル」タブを選択し、紐づいているルートテーブルのリンクをクリックする。
4. ルートテーブルの画面で「ルート」タブを選択し、右側の「ルートを編集」をクリックする。
5. 画面左下の「ルートを追加」をクリックし、以下を入力する。
   - **送信先**：`0.0.0.0/0`
   - **ターゲット**：`インターネットゲートウェイ` を選択 → 表示された入力欄で Step 3 で作成した IGW を選択
6. 画面右下の「変更を保存」をクリックする。

> **確認：** ルートテーブルの「ルート」タブに `0.0.0.0/0 → <IGW のID>` の行が追加されていることを確認する。

------------------------------

### Step 5：EC2 インスタンスの作成

**目的：** 構築したネットワーク上に EC2 インスタンス（部屋の中に配置する人＝サーバ）を起動する。

#### 操作手順

1. AWS マネジメントコンソール上部の検索欄に「EC2」と入力し、EC2 サービスを選択する。
2. オレンジ色の「インスタンスを起動」をクリックする。
3. 以下の項目を入力する。
   - **名前**：`<任意の名前>_ec2`
   - **AMI**：Amazon Linux 2023
   - **インスタンスタイプ**：`t2.micro`
   - **キーペア**：
     - 既存のキーペアがある場合 → 該当のキーペアを選択
     - ない場合 → 「新しいキーペアの作成」をクリックして作成（ダウンロードした `.pem` ファイルは大切に保管）
4. 「ネットワーク設定」の右側にある「編集」をクリックし、以下を設定する。
   - **VPC**：Step 1 で作成した VPC
   - **サブネット**：Step 2 で作成したサブネット
   - **パブリック IP の自動割り当て**：`有効化`
   - **ファイアウォール（セキュリティグループ）**：「セキュリティグループを作成」を選択
     - **セキュリティグループ名**：`<任意の名前>_sg`
     - **説明**：`Allow SSH from my IP`
     - **タイプ**：`SSH`
     - **ソースタイプ**：`マイIP`
5. 画面右下の「インスタンスを起動」をクリックする。

> **確認：**
> - EC2 インスタンス一覧でステータスが「実行中」、ステータスチェックが「2/2 のチェックに合格」になっていることを確認する。
> - 「パブリック IPv4 アドレス」が割り当てられていることを確認する。
> - ローカル PC のターミナルから以下のコマンドで SSH 接続できることを確認する。
>   ```
>   ssh -i <キーペアファイル.pem> ec2-user@<パブリックIPアドレス>
>   ```

### C. 用語解説

| 用語 | 説明 |
|------|------|
| VPC | Virtual Private Cloud の略。AWS 上に作成する自分専用の仮想ネットワーク空間。 |
| サブネット | VPC を細分化した小さなネットワーク。アベイラビリティーゾーン単位で作成する。 |
| CIDR | Classless Inter-Domain Routing。IP アドレスの範囲を `10.1.0.0/16` のように表す表記法。 |
| インターネットゲートウェイ（IGW） | VPC とインターネットを接続するための AWS のコンポーネント。 |
| ルートテーブル | サブネット内の通信経路を定義する表。 |
| EC2 | Elastic Compute Cloud。AWS 上の仮想サーバ。 |
| セキュリティグループ | EC2 インスタンスに対する仮想的なファイアウォール。 |
| キーペア | SSH 接続時に使用する公開鍵と秘密鍵のペア。 |

------------------------------

### D. 補足解説

- **なぜサブネットマスクが `/18` なのか**
  VPC の CIDR `10.1.0.0/16` を 4 人で分割するため、ホスト部から 2 ビット分をネットワーク部に追加する。`16 + 2 = 18` のため `/18` となる。
  - メンバー A：`10.1.0.0/18`
  - メンバー B：`10.1.64.0/18`
  - メンバー C：`10.1.128.0/18`
  - メンバー D：`10.1.192.0/18`

- **パブリックサブネットとプライベートサブネットの違い**
  ルートテーブルに `0.0.0.0/0 → IGW` の経路が設定されているサブネットを「パブリックサブネット」、設定されていないサブネットを「プライベートサブネット」と呼ぶ。本手順書で作成するのはパブリックサブネット。
