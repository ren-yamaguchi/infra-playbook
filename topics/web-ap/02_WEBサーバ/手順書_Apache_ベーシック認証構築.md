## 2. 目的・概要

### 2-1. 目的

> 本手順書では，AWSのEC2を1台用いてApacheでベーシック認証を導入したWebサーバーの構築手順について説明する．
> 構築後はIPアドレスで検索した時にベーシック認証をクリアした場合のみページが表示されることを目指す．

### 2-2. 構成概要（アーキテクチャ）


### 2-3. 完成イメージ（ゴール定義）

- ブラウザから `http://<パブリックIP>` にアクセスできる
- アクセスした後，認証をクリアすることでWebページを閲覧できる

### Step 1 システム設定

**目的：** システムの変更と更新を行う

#### 操作手順

```bash
# ローカルPCからsshログイン
ssh -i <秘密鍵のファイルパス> ec2-user@<EC2のパブリックIP>

# rootにユーザーにスイッチ
sudo su -

# 最新パッケージの更新
dnf update -y

# システムの時間を日本時間に設定(これによりログを確認した時の時間が日本時間で表示されるようになる)
timedatectl set-timezone Asia/Tokyo
```

------------------------------

### Step 2 Apacheの設定

**目的：** Webサーバー構築のためのApacheの設定を行う

#### 操作手順

```bash
# Apacheの最新バージョンをインストール
dnf install -y httpd

# Apacheの起動と自動起動設定
systemctl enable --now htppd

# Apacheの状態確認
systemctl status httpd | less

# Apacheの自動起動設定の確認
systemctl is-enabled httpd
```

> **確認：** Webサーバが機能確認
> ブラウザでEC2パブリックIPアドレスを入力し，ブラウザ画面で **「IT WORKS!」** と表記されれば成功

### Step 3 ベーシック認証のパスワードファイルの作成

**目的：** ユーザー名と暗号化されたパスワードを保存するファイルを作成

#### 操作手順

```bash
# パスワードファイルの作成とユーザーの追加
htpasswd -c /etc/httpd/.htpasswd <ユーザー名>

	#---以下のようにパスワードを求められるので設定するパスワードを入力
	New password:
	Re-type new password:
	#-----------------------------------------------------------

# ファイルの所有ユーザーと所有グループを変更
chown apache:apache /etc/httpd/.htpasswd

# ファイルの権限を変更
chmod 600 /etc/httpd/.htpasswd
```

### Step 4 Apache設定ファイルの編集

**目的：** 認証を適用するディレクトリを指定し，認証設定を追記

#### 操作手順

```bash
# Apacheの設定ファイルのバックアップ取得（原本保存）
cp /etc/httpd/conf/httpd.conf{,.org}

# 設定ファイルに追記
vi /etc/httpd/conf/httpd.conf

	#---下記を設定ファイルの一番下に追記----
	<Directory "/var/www/html/<ディレクトリ名>">
		AuthType Basic
		AuthName "Restricted Area"
		AuthUserFile /etc/httpd/.htpasswd
		Require valid-user
	</Directory>
	#------------------------------------

# 設定ファイルの構文チェック
apachectl configtest

	#---成功例
	Syntax OK
	#--------

# 設定を反映させるため再起動
systemctl restart httpd
```

## 付録

### A. コマンド解説

#### A-1. htpasswdコマンド

`htpasswd <オプション> <ファイルパス> <ユーザー名>`：`htpasswd`コマンドは，Webサーバー（主にApache）でBasic認証（基本認証）に使用する「ユーザー名」と「暗号化パスワード」の組み合わせを管理するコマンドです．
- `htpasswd`
  - 役割：パスワードファイルを操作する基本コマンド
  - 動作：実行後に設定するパスワードの入力を求められます
- `<オプション>`：コマンドの挙動を切り替えます．省略も可能ですが，状況に応じて以下の主要なオプションを指定します．
  - `-c`：ファイルを新規作成します．既存ファイルがある場合は上書き削除されるため注意してください．
  - `-B`：パスワードの暗号化に，より安全な `bcrypt` アルゴリズムを使用します．
  - `-D`：指定したユーザーをファイルから削除します．
  - ※既存のファイルにユーザーを「追加・更新」する場合は，オプションを何も付けずに実行します．
- `<ファイルパス>`
  - 役割：認証情報を保存する（または保存されている）パスワードファイルの場所とファイル名の指定です．
  - 慣例：一般的に `.htpasswd` というファイル名が使われます．例: `/var/www/.htpasswd` 
- `<ユーザー名>`
  - 役割：新規登録，あるいはパスワードを変更したい認証用ユーザーの識別名（ID）です．
  - 仕様：WebブラウザでBasic認証画面が出た際に，入力するユーザー名（ID）になります．

------------------------------

#### A-2. パスワードファイルの所有権変更

```bash
chown apache:apache /etc/httpd/.htpasswd
chmod 600 /etc/httpd/.htpasswd
```
**目的：** 上記のコマンドを実行する理由について説明する

上記のコマンドはパスワードファイルの所有ユーザーと所有グループを変更し，権限を変更するコマンドなのだが，この設定を怠ると，**サーバー内部の他のユーザーやプロセスからパスワードを盗まれるリスク**が高まる．

- ハッシュ化されたパスワードの漏洩を防ぐ
  `.htpasswd`に書き込まれているパスワードは暗号化（ハッシュ化）されている．
  しかし，ファイル自体の読み取り権限（`r`）が全員にあると，悪意のある一般ユーザーや，他の脆弱なWebアプリケーション（PHPのファイル読み込み脆弱性など）を介して，ハッシュ化された文字列を丸ごと盗まれる危険がある

- 総当たり攻撃（ブルートフォース）の標的になる
  盗まれたハッシュ文字列を攻撃者が手元のパソコンに持ち帰ると，高性能なマシンを使って高速にパスワードを解析（クラック）されてしまう．

------------------------------

### B. 設定ファイル解説

#### B-1. Apache設定ファイル（ベーシック認証内容）

```bash
<Directory "/var/www/html/<ディレクトリ名>">
	AuthType Basic
	AuthName "Restricted Area"
	AuthUserFile /etc/httpd/.htpasswd
	Require valid-user
</Directory>
```

**設定内容：** Apacheウェブサーバーで特定のディレクトリにベーシック認証（基本認証）を設定するための記述です．
- `<Directory "/var/www/html/<ディレクトリ名>">`
  - 指定したパスのディレクトリに対して，これ以降の設定を適用することを宣言します．
  - `<ディレクトリ名>`の部分には，制限したい実際のフォルダ名が入ります．
- `AuthType Basic`
  - 認証の方式として「ベーシック認証（基本認証）」を使用することを指定します．
  - ブラウザが標準で持つIDとパスワードの入力ダイアログを表示させます．
- `AuthName "Restricted Area"`
  - 認証画面（ダイアログ）に表示する設定名称（レルム）を指定します．
  - 画面には「Restricted Area（制限されたエリア）」というテキストが表示されます．
- `AuthUserFile /etc/httpd/.htpasswd`
  - アクセスを許可するユーザー名と，暗号化されたパスワードが保存されているファイルのパスを指定します．
  - ここでは `/etc/httpd/.htpasswd` というファイルを参照しています．
- `Require valid-user`
  - パスワードファイル（`.htpasswd`）に登録されているすべての有効なユーザーに対してアクセスを許可します．
  - 正しいIDとパスワードを入力すれば誰でも閲覧可能になります．
- `</Directory>`
  - `<Directory>` で始まった設定の範囲（ブロック）が，ここで終了することを示します．
