## 2. 目的・概要

### 2-1. 目的

> 本手順書では、AWSのEC2を1台用いてNginxでWebサーバーの構築手順について説明する。
> 構築後はIPアドレスで検索した時にNginxのデフォルトページが表示されることを目指す。

### 2-2. 構成概要（アーキテクチャ）


### 2-3. 完成イメージ（ゴール定義）

- ブラウザから `http://<パブリックIP>` にアクセスできる

## 4. 構築手順（詳細）

> **注意事項**
> - コマンド中の `<山カッコ>` は自分の環境の値に置き換えること

------------------------------

### Step 1 システム設定

**目的：** システムの変更と更新を行う

#### 操作手順

```bash
# ローカルPCからsshログイン
ssh -i <秘密鍵のファイルパス> ec2-user@<EC2のパブリックIP>

# rootにユーザーにスイッチ
sudo su -

# 最新パッケージの更新
dnf update -y

# システムの時間を日本時間に設定(これによりログを確認した時の時間が日本時間で表示されるようになる)
timedatectl set-timezone Asia/Tokyo
```

------------------------------

### Step 2 Nginxの設定

**目的：** Webサーバー構築のためのNginxの設定を行う

#### 操作手順

```bash
# Apacheの最新バージョンをインストール
dnf install -y nginx

# Apacheの起動と自動起動設定
systemctl enable --now nginx

# Apacheの状態確認
systemctl status nginx | less

# Apacheの自動起動設定の確認
systemctl is-enabled nginx
```

**確認：** Webサーバの機能確認
ブラウザで「`http://EC2パブリックIPアドレス`」を入力し、ブラウザ画面で **「Welcome to nginx!」** と表記されれば成功
