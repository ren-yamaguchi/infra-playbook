## 2. 目的・概要

### 2-1. 目的

> 本手順書では、AWSのEC2を1台用いてApacheを用いたWebサーバーの構築手順について説明する。
> 構築後はブラウザでEC2のパブリックIPアドレスを入力し、ブラウザ画面で **「IT WORKS!」** と表記される状態を目指す。

### 2-2. 構成概要（アーキテクチャ）

### 2-3. 完成イメージ（ゴール定義）

- ブラウザから `http://<パブリックIP>` にアクセスできる
- アクセスした後、認証をクリアすることでWebページを閲覧できる

## 4. 構築手順（詳細）

> **注意事項**
>
> - コマンド中の `<山カッコ>` は自分の環境の値に置き換えること
>

------------------------------

### Step 1 システム設定

**目的：** システムの変更と更新を行う

#### 操作手順（システム設定）

```bash
# --- ローカルPC ---

# ローカルPCからsshログイン
ssh -i <秘密鍵のファイルパス> ec2-user@<EC2のパブリックIP>

# --- EC2 にログイン後 ---

# rootにユーザーにスイッチ
sudo su -

# 最新パッケージの確認
dnf update -y

# 最新パッケージの更新
dnf upgrade -y

# システムの時間を日本時間に設定(これによりログを確認した時の時間が日本時間で表示されるようになる)
timedatectl set-timezone Asia/Tokyo
```

------------------------------

### Step 2 Apacheの設定

**目的：** Webサーバー構築のためのApacheの設定を行う

#### 操作手順（Apacheの設定）

```bash
# Apacheの最新バージョンをインストール
dnf install -y httpd

# Apacheの起動と自動起動設定
systemctl enable --now httpd

# Apacheの状態確認
systemctl status httpd | less

# Apacheの自動起動設定の確認
systemctl is-enabled httpd
```

>
> **確認：** Webサーバが機能確認
> 
> ブラウザでEC2パブリックIPアドレスを入力し、ブラウザ画面で **「IT WORKS!」** と表記されれば成功
>
